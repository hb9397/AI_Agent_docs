# 화면 구현 함정 체크리스트 (pitfall-checklist)

구현 지침서 초안 완성 후 출력 전에 아래를 자체 점검한다.

---

## FE 컴포넌트 함정

```
[ ] 컴포넌트 책임이 과도하지 않은가? — 1개 컴포넌트가 API 호출 + 렌더링 + 상태 관리를 모두 하면 분리
[ ] Props drilling이 3단계 이상인가? — Context 또는 상태 관리 라이브러리 도입 검토
[ ] 리렌더링 폭탄이 없는가? — 부모 상태 변경 시 불필요한 자식 렌더링
[ ] 폼 유효성 검사가 클라이언트/서버 양쪽에 있는가? — 한쪽만 있으면 위험
[ ] 파일 업로드 화면에서 용량 제한이 명시되어 있는가?
```

---

## API 연동 함정

```
[ ] 목록 조회에 페이징이 있는가? — 없으면 대용량에서 성능 문제
[ ] API 응답 형식이 화면 필요와 일치하는가? — 변환 레이어 필요 여부
[ ] 낙관적 업데이트(optimistic update)가 필요한 곳이 있는가?
[ ] API 호출 중복 방지가 있는가? — 더블 클릭, 빠른 필터 변경 시
[ ] 인증 토큰 만료 시 자동 갱신 처리가 있는가?
```

---

## 상태 관리 함정

```
[ ] 서버 상태와 로컬 상태가 혼재되어 있지 않은가?
[ ] 화면 이동 후 돌아왔을 때 상태가 유지되어야 하는가? — 필터, 스크롤 위치
[ ] 모달 안의 폼 데이터가 모달 닫힘 시 초기화되는가?
[ ] URL과 화면 상태가 동기화되어야 하는가? — 필터 공유, 북마크 지원
```

---

## 화면 구조 함정

```
[ ] 화면 간 공통 레이아웃이 중복 구현되어 있지 않은가?
[ ] 라우팅 가드(인증 체크)가 누락된 화면이 없는가?
[ ] SFR 대응 누락 화면이 없는가? — 매핑 표와 대조
[ ] 화면 이름이 실제 기능과 일치하는가? — 혼동 방지
[ ] 구현 순서대로 했을 때 이전 Phase 없이 작동하지 않는 화면이 없는가?
```
