# basic-instruction.md 추출 기준 (analysis-instruction)

설계 문서에서 basic-instruction.md 생성에 필요한 항목만 추출한다.

---

## 추출 항목

### 1. 아키텍처 제약
- 레이어 간 의존 방향 (어디서 어디를 호출할 수 있는가)
- 집중 관리 모듈 (API 클라이언트, 훅 등 — 직접 호출 금지 대상)
- 컴포넌트 역할 경계 (비즈니스 로직 위치)

### 2. 코딩 컨벤션
- 함수/컴포넌트 작성 스타일
- 비동기 처리 방식 (try/catch 필수 여부)
- 상태 관리 방식

### 3. 금지 목록 — 반드시 삼위일체로 추출
각 금지 항목은 아래 세 가지를 반드시 함께 추출한다.
하나라도 빠지면 Agent가 왜 금지인지 모르고 우회 코드를 생성한다.

```
금지 패턴  →  금지 이유  →  대안
```

예시 형태:
- `axios 직접 import` → API 호출 분산 시 유지보수 불가 → `src/api/client.js` 함수 사용
- `window.location.href` → SPA 라우터 상태 초기화됨 → `useNavigate()`

### 4. 프레임워크별 규칙
- React 규칙 (key prop, hook 의존성 배열 등)
- 라우터 규칙
- 스타일링 규칙 (인라인 style 금지 여부 등)

### 5. 특수 플로우 규칙
- 순서가 고정된 플로우가 있는가 (온보딩, 결제 등)
- 있으면 단계별 트리거 + API 매핑 추출

### 6. 파일 생성 규칙
- 새 파일을 어느 위치에 만드는가
- 네이밍 컨벤션

### 7. AI Agent 전용 규칙
- Agent가 코드 작성 시 반드시 지켜야 할 것
- Agent가 해서는 안 되는 것 (예: 주석 작성 금지)

## 누락 항목 처리

항목이 없으면 `미정 — [이유]` 로 표시한다.
사용자에게 확인이 필요한 경우 최대 1개만 묻는다. (analysis-claude에서 이미 질문했으므로)

🔴 필수: 금지 목록 이유/대안이 없으면 반드시 확인
