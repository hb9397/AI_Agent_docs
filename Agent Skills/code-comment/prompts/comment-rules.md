# 주석 작성 규칙 (comment-rules)

모든 주석은 **한글**로 작성한다.

---

## 0. 이모지 사용 원칙

이모지는 **절대 남발하지 않는다.**

허용 대상: 개발자가 반드시 인지해야 하는 중요 경고·주의 지점 1곳에만 사용한다.
- `⚠️` 사이드이펙트, 호출 순서 강제, 전역 상태 직접 변경 등 실수 유발 가능 지점
- `🔒` 인증·권한·보안 관련 처리 (무단 제거 금지 의미 전달)

금지 대상: 파일 요약, 메서드 설명, 변수 주석, 화면 요소, 변경 이력 등 일반 주석에는 사용하지 않는다.
단순 강조나 가독성 목적의 이모지 사용은 금지한다.

---

## 1. 파일 최상단 블록 주석

파일의 가장 첫 줄에 작성한다. 언어별 블록 주석 형식 사용.
아래 4가지 항목을 3~5줄로 서술한다.

- 파일명
- 이 파일이 속한 기능 영역
- 이 파일의 역할
- 주요 export / 진입점 (있는 경우)

---

## 2. 클래스 · 메서드 · 프레임워크 요소

대상: `class`, `function`, `const fn = () =>`, React Hook (`useXxx`), Type/Interface, Decorator 등

**상단 블록 주석** (3~5줄)
- 무슨 기능인지
- 어디서 사용되는지 (역할)
- 주요 파라미터 또는 반환값 (있는 경우)

**내부 로직 주석**
- 핵심 분기·처리 흐름을 단계별로 한 줄씩 서술
- 복잡한 조건문·반복문 앞에 의도 설명
- 외부 API·DB 호출 지점에 목적 명시

---

## 3. 변수 · 상수

코드 우측에 인라인 주석으로 짧게 작성한다.
내용이 자명한 변수(`i`, `idx` 등 루프 변수)는 생략해도 된다.

---

## 4. HTML · JSX 화면 요소

대상: `<div>`, `<section>`, `<form>`, `<button>`, `<select>` 등 UI 구성 요소

해당 요소 바로 위에 한 줄 주석으로 서술한다.
- 어떤 화면/섹션을 구성하는지
- 버튼·입력 요소면 어떤 동작을 트리거하는지

---

## 5. 파일 최하단 변경 이력

**신규 파일(git 이력 없음)** → 이 섹션 전체 생략

**기존 파일** → 최신 변경 1건만 유지 (누적 금지)

수집 방법:
```bash
# 마지막 커밋 정보
git log -1 --format="%ae %ad %s" --date=format:"%Y-%m-%d" -- <파일경로> 2>/dev/null

# 현재 변경사항 요약
git diff HEAD -- <파일경로> 2>/dev/null | head -60
```

작성자를 git에서 찾지 못하면 사용자에게 직접 묻는다.
