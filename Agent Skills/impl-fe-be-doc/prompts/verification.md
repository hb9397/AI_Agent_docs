# 검증 시나리오 작성 규칙 (verification)

---

## 통합 검증 형식

```markdown
### ✅ Phase N 통합 검증

**시나리오**: [한 줄 요약 — 무엇을 확인하는 Phase인지]

1. [URL 접속 또는 명령 실행]
2. [사용자 액션]
3. [기대 동작 확인]
4. [DB 또는 로그에서 확인]

**합격 조건** : [조건 A] ✅ / [조건 B] ✅ / [조건 C] ✅
```

---

## 좋은 검증 vs 나쁜 검증

```
✅ 좋은 검증
  "DB에서 SELECT * FROM sites; 실행 — 레코드 존재 확인"
  "LogViewer에 SUCCESS 타입 로그 실시간 표시 확인"
  "VITE_NOVNC_URL 미설정 시 텍스트 안내, 설정 시 iframe 렌더링"

❌ 나쁜 검증
  "정상적으로 동작하는지 확인"
  "에러 없이 실행되는지 확인"
  "BE가 잘 동작하는지 확인"
```

---

## 합격 조건 개수 기준

- 단순 Phase: 2~3개
- 핵심 기능 Phase: 4~5개
- 배포 Phase: 서비스별 1개씩
