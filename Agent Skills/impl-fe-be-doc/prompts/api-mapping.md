# API 연동 포인트 규칙 (api-mapping)

화면 중심 Phase에서 화면별 API 호출 지점과 BE 태스크 범위를 연결할 때 사용한다.

---

## 화면별 API 매핑 형식

```markdown
### API 연동

| # | 메서드 | 경로 | 호출 시점 | 요청 | 응답 | 에러 처리 |
|---|--------|------|----------|------|------|----------|
| 1 | GET | /api/inspections | 화면 마운트 + 필터 변경 | query: page, size, status, keyword | { items: [], total: N } | 에러 토스트 + 재시도 |
| 2 | GET | /api/inspections/{id} | 행 클릭 | path: id | 상세 데이터 | 에러 토스트 |
| 3 | POST | /api/inspections | 등록 폼 제출 | body: 필드 값 | { id: N } | 유효성 에러 → 필드별 표시 |
```

---

## API 호출 시점 분류

```text
마운트 시: 화면 진입 시 자동 호출 (목록 조회, 초기 데이터)
사용자 액션: 클릭, 폼 제출, 필터 변경 등
자동 갱신: 폴링, WebSocket 이벤트, Focus 복귀 시
조건부: 특정 상태일 때만 호출 (권한 체크 후, 데이터 존재 시)
```

---

## 에러 처리 패턴

| HTTP 상태 | 처리 방식 |
|-----------|----------|
| 400 Bad Request | 유효성 에러 → 필드별 인라인 에러 메시지 |
| 401 Unauthorized | 로그인 페이지로 리다이렉트 |
| 403 Forbidden | 권한 없음 안내 + 이전 화면 또는 안전한 화면으로 이동 |
| 404 Not Found | 데이터 없음 안내 + 목록으로 이동 |
| 409 Conflict | 충돌 안내 다이얼로그 + 재시도 옵션 |
| 500 Server Error | 에러 토스트 + 재시도 버튼 |
| Network Error | 오프라인 안내 + 자동 재시도 |

---

## BE 태스크 연결 규칙

- 화면 API 표에 있는 신규/변경 API는 반드시 같은 Phase의 `BE-XX` 태스크에 반영한다.
- API가 이미 존재하는 경우 `FE-XX` 태스크의 의존을 해당 기존 API 확인 또는 수정 태스크로 명시한다.
- 요청/응답 스키마가 화면 요구와 맞지 않으면 DTO/adapter/mapper 태스크를 추가한다.
