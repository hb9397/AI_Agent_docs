# 스케일 라우팅 (scale-routing)

스케일 선택에 따라 사용할 INPUT / OUTPUT 양식과 활성화할 섹션을 결정한다.

---

## 스케일별 양식 매핑

| 스케일 | INPUT 양식 | OUTPUT 양식 | 비고 |
|--------|-----------|------------|------|
| 프로젝트 전체 | INPUT_V1 | OUTPUT_V1 | Agent 스킬에서는 지양. 사용자가 명시적으로 요청 시에만 허용 |
| 화면 단위 | INPUT_V2 | OUTPUT_V2 | 기본 권장 스케일 |
| 기능 단위 | INPUT_V2 | OUTPUT_V2 | |
| 컴포넌트 / 로직 단위 | INPUT_V2 | OUTPUT_V2 | |

---

## 섹션 활성화 규칙

### 🔵 모든 스케일 공통
- A. 한 줄 요약
- B. 문제와 목적
- D. 범위
- E. 동작 흐름
- G. 집중할 로직
- H. 뒤집기 확인
- I. 열린 질문

### 🟢 프로젝트 전체 전용
- C-1. 사용자와 환경
- F-1. 기술 스택
- OUTPUT 07. 라이브러리 및 외부 구성
- OUTPUT 11. 부가 정보

### 🟠 화면 / 기능 / 컴포넌트 전용
- C-2. 기존 코드 컨텍스트
- F-2. 인터페이스 경계
- OUTPUT 04. 인터페이스 설계
- OUTPUT 08. 엣지 케이스 & 에러 처리
- OUTPUT 09. 테스트 시나리오

---

## 스케일 판단 가이드

사용자 발화가 모호한 경우 아래 기준으로 판단하여 확인한다.

| 발화 예시 | 판단 스케일 |
|-----------|------------|
| "온보딩 화면 설계해줘" | 화면 단위 |
| "로그인 기능 설계해줘" | 기능 단위 |
| "이 컴포넌트 어떻게 짜면 돼?" | 컴포넌트 단위 |
| "프로젝트 전체 설계해줘" | 프로젝트 전체 (사용자 재확인 권고) |
