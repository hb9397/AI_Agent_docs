# 범용 함정 체크리스트 (pitfall-checklist)

작업지침서 초안 완성 후 출력 전에 아래를 자체 점검한다.
**프로젝트 유형에 따라 해당하는 섹션만 실행한다.**

---

## 공통 함정

```
[ ] Phase가 너무 큰가? — 한 Phase에 모듈 5개 이상이면 쪼갠다
[ ] 각 Phase 끝에 확인 가능한 결과물이 있는가?
[ ] 태스크 의존 관계가 순환하는가? — A→B→A 형태 금지
[ ] 미결 사항이 구현을 막는가? — 결정 필요 항목은 해당 태스크 전에 표시
[ ] 외부 의존성 설치 누락이 없는가? — INIT Phase에 포함 확인
[ ] .gitignore 대상이 명시되어 있는가? — 시크릿, 빌드 산출물, 캐시
```

---

## 입출력 함정

```
[ ] CLI args 파싱 에러 처리가 있는가? — 잘못된 옵션, 필수 옵션 누락
[ ] 파일 I/O 에러 처리가 있는가? — 파일 없음, 권한 없음, 디스크 부족
[ ] 인코딩 처리가 명시되어 있는가? — UTF-8 / EUC-KR 혼재 가능성
[ ] 대용량 입력 처리 전략이 있는가? — 스트리밍 vs 전체 로드
[ ] stdout / stderr 분리가 되어 있는가? — 정상 출력 vs 에러 메시지
```

---

## 외부 연동 함정

```
[ ] API 호출에 타임아웃·재시도 전략이 있는가?
[ ] 인증 토큰 / API 키가 환경변수로 분리되는가? — 하드코딩 금지
[ ] Rate limit 대응이 있는가? — 외부 API 호출 빈도 제한
[ ] 네트워크 에러 시 그레이스풀 처리가 있는가?
[ ] 외부 시스템 다운 시 폴백이 있는가?
```

---

## 테스트 함정

```
[ ] 핵심 로직에 단위 테스트가 있는가? — CORE-XX 태스크에 대응
[ ] 테스트 데이터(fixture)가 정의되어 있는가?
[ ] 테스트가 외부 시스템에 의존하지 않는가? — mock/stub 사용 여부
[ ] 테스트 실행 명령이 명확한가? — 검증 시나리오에 포함
```

---

## 패키징 함정

```
[ ] 빌드 스크립트가 있는가? — `npm run build` / `python setup.py` 등
[ ] 의존성 잠금 파일이 있는가? — package-lock.json / poetry.lock
[ ] 실행 권한 설정이 필요한가? — shebang, chmod +x
[ ] README / 사용법 문서가 포함되어 있는가?
[ ] 버전 관리 방식이 정해져 있는가? — semver, 날짜 기반 등
```
