# 범용 태스크 작성 규칙 (task-rules)

---

## 태스크 ID 체계

```
INIT-XX : 초기화 (프로젝트 구조, 설정, 의존성 설치)
CORE-XX : 핵심 로직 (비즈니스 로직, 알고리즘, 핵심 모듈)
IO-XX   : 입출력 (CLI 파싱, 파일 I/O, API 클라이언트, DB 접근)
TEST-XX : 테스트 (단위 테스트, 통합 테스트, 평가)
PKG-XX  : 패키징 (빌드, 배포, 문서화, CI/CD)
```

번호는 Phase 내 순서가 아닌 **전체 문서 통합 일련번호**로 부여한다.
(예: Phase 1에서 CORE-01~03, Phase 2에서 CORE-04~06으로 이어짐)

---

## 태스크 작성 형식

```markdown
### [태스크 ID] · [태스크 이름] (`파일 경로`)

**의존** : [선행 태스크 ID 목록. 없으면 "없음"]

[구현 내용 — 무엇을 왜 만드는지]

**Agent 지시** : (있을 때만 포함)
[핵심 함수 시그니처, 중요 분기 처리, 설계 결정 이유]

**검증 기준** : [이 태스크 단독 완료 판단 조건]
```

## 프로젝트 유형별 태스크 ID 사용 가이드

### CLI 도구
```
INIT-01: package.json / pyproject.toml 초기화
IO-01:   CLI args 파서 (yargs, argparse, cobra 등)
CORE-01: 핵심 처리 함수
IO-02:   출력 포매터 (JSON, 테이블, 컬러 등)
TEST-01: 핵심 기능 단위 테스트
PKG-01:  빌드 스크립트 + 바이너리 패키징
```

### 자동화 스크립트
```
IO-01:   데이터 소스 커넥터 (API 클라이언트, 크롤러)
CORE-01: 데이터 변환/처리 로직
IO-02:   결과 저장 (파일, DB, API 전송)
CORE-02: 스케줄링 / 재시도 로직
TEST-01: 파이프라인 통합 테스트
```

### 라이브러리
```
INIT-01: 프로젝트 구조 + 빌드 설정
CORE-01: 코어 API (public interface)
CORE-02: 내부 유틸리티
TEST-01: API별 단위 테스트
PKG-01:  NPM / PyPI 패키징 + README
```

## Agent 지시 작성 기준

포함할 것:
- 핵심 함수/클래스 시그니처
- 의도적인 설계 결정 이유
- 환경별 분기 처리 (개발/배포, OS별 등)
- 주의해야 할 엣지 케이스
- 외부 라이브러리 사용법 힌트 (처음 쓰는 기술인 경우)

포함하지 말 것:
- 완성된 구현 코드 전체 (작성 여지를 남긴다)
- 표준 보일러플레이트
- 이미 다른 태스크에서 설명한 내용 반복
