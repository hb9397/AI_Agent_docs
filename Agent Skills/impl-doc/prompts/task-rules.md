# 태스크 작성 규칙 (task-rules)

---

## 태스크 ID 체계

```
INF-XX : 인프라 (디렉토리 구조, 컨테이너, Dockerfile 등)
BE-XX  : 백엔드
FE-XX  : 프론트엔드
```

번호는 Phase 내 순서가 아닌 **전체 문서 통합 일련번호** 로 부여한다.
(예: Phase 1에서 BE-01~04, Phase 2에서 BE-05~06으로 이어짐)

---

## 태스크 작성 형식

```markdown
### [태스크 ID] · [태스크 이름] (`파일 경로`)

**의존** : [선행 태스크 ID 목록. 없으면 "없음"]

[구현 내용 — 무엇을 왜 만드는지]

**Agent 지시** : (있을 때만 포함)
[핵심 함수 시그니처, 중요 분기 처리, 설계 결정 이유]

**검증 기준** : [이 태스크 단독 완료 판단 조건]
```

---

## Agent 지시 작성 기준

포함할 것:
- 핵심 함수 시그니처 / 클래스 구조
- 중요한 환경변수 처리 방식 (개발/배포 분기 등)
- 의도적인 설계 결정 이유
- 주의해야 할 분기 처리

포함하지 말 것:
- 완성된 구현 코드 전체 (작성 여지를 남긴다)
- 표준 보일러플레이트 (import, 단순 초기화 등)
- 이미 다른 태스크에서 설명한 내용 반복
