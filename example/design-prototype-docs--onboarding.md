# 목업 디자인 문서: ACRO 사이트 온보딩

## 1. 기본 정보

| 항목 | 내용 |
|---|---|
| **요구사항 번호** | REQ-ONB-01 |
| **요구사항 명칭** | 예약 사이트 온보딩 (1회 시연 기반 셀렉터 등록) |
| **프로젝트명** | ACRO — 범용 예약 매크로 |
| **메인 색상** | `#1E3A8A` (딥 블루) |
| **화면 수** | 3개 |

## 2. 요구사항 요약

사용자가 새 예약 사이트를 ACRO에 처음 등록하는 흐름이다. 사용자가 브라우저에서 직접 로그인 후 예약 화면의 요소를 한 번씩 클릭하면, AI가 각 요소의 의미를 추론하고 셀렉터를 저장한다. 이후 매크로가 저장된 셀렉터로 자동 예약을 수행한다.

세부 기능:
1. 사이트 기본 정보 등록 (`site_name`, 예약 URL)
2. 사용자 직접 로그인 + 예약 화면 요소 시연·캡처
3. AI 추론 결과(요소명·셀렉터) 확인 및 검증 후 저장

## 3. 화면 목록

| 화면 번호 | 화면명 | HTML | CSS | JS | JSON | 레이아웃 패턴 |
|---|---|---|---|---|---|---|
| 1 | 사이트 등록 | `display/ONB-001-site-register.html` | `css/ONB-001-site-register.css` | `script/ONB-001-site-register.js` | `data/ONB-001-site-register-data.json` | 폼 화면 |
| 2 | 요소 시연·캡처 | `display/ONB-002-capture.html` | `css/ONB-002-capture.css` | `script/ONB-002-capture.js` | `data/ONB-002-capture-data.json` | 앱 셸 |
| 3 | 셀렉터 검증·저장 | `display/ONB-003-review.html` | `css/ONB-003-review.css` | `script/ONB-003-review.js` | `data/ONB-003-review-data.json` | 마스터-디테일 |

## 4. 화면 간 흐름

```text
[온보딩 마법사 3단계]
  ① 사이트 등록   ② 요소 시연·캡처   ③ 셀렉터 검증·저장

[화면 간 연결]
  사이트 등록(1) → ([다음] 클릭, site_name·URL 전달) → 요소 시연·캡처(2)
  요소 시연·캡처(2) → (요소 N개 캡처 완료 후 [검토] 클릭, 캡처 목록 전달) → 셀렉터 검증·저장(3)
  셀렉터 검증·저장(3) → ([수정] 클릭) → 요소 시연·캡처(2) 로 복귀
  셀렉터 검증·저장(3) → ([저장] 클릭) → 온보딩 완료 → 매크로 관리 화면(외부)
```

## 5. 화면별 상세

### 화면 1 — 사이트 등록 (폼 화면)

**5-1. 목적** — 새 예약 사이트를 식별자(`site_name`)와 예약 URL로 등록한다. REQ-ONB-01 세부 1 담당.

**5-2. 기능 + 이유**

| 기능 | 이유 |
|---|---|
| `site_name` 입력 + 중복 검사 | 사이트별 셀렉터를 독립 관리하므로 고유 식별자가 먼저 필요 |
| 예약 URL 입력 + 형식 검증 | 다음 단계에서 이 URL로 브라우저를 띄우므로 유효성 선검증 |
| 모의 사이트 토글 | 실제 코레일 대신 MOCK-TRAIN으로 안전하게 기능 검증 가능 |

**5-3. 기능 흐름**

```text
[화면 1 — 사이트 등록 흐름]
  site_name 입력 → 중복 검사(실시간)
       │
       ▼
  예약 URL 입력 → 형식 검증
       │
       ▼
  (선택) 모의 사이트 토글 ON → URL 기본값 자동 채움
       │
       ▼
  [다음] 버튼 활성화 → 화면 2로 이동
```

**5-4. 레이아웃 구조**

```text
[중앙 정렬 카드 — 최대 560px]
  제목: "새 예약 사이트 등록"  부제: "1/3 단계"
  • site_name 입력칸 (우측 중복 검사 상태 아이콘)
  • 예약 URL 입력칸
  • 모의 사이트 사용 토글
[카드 하단 우측]
  [다음] 버튼 (검증 통과 전 비활성)
```

**5-5. 주요 컴포넌트** — StepIndicator, TextField(검증 상태), ToggleSwitch, PrimaryButton

**5-6. 더미 데이터**

```json
{ "site_name": "코레일-예매", "reserve_url": "https://www.letskorail.com/...", "use_mock": false, "name_check": "available" }
```

### 화면 2 — 요소 시연·캡처 (앱 셸)

**5-1. 목적** — 사용자가 실제 예약 화면에서 요소를 클릭하면 AI가 의미를 추론한다. REQ-ONB-01 세부 2 담당.

**5-2. 기능 + 이유**

| 기능 | 이유 |
|---|---|
| 임베디드 브라우저(noVNC) 패널 | 사용자가 직접 로그인·클릭해야 실제 세션과 요소를 캡처 가능 |
| 클릭 캡처 → AI 추론 결과 실시간 표시 | 어떤 요소가 무엇으로 인식됐는지 즉시 확인해야 오인식 교정 가능 |
| 캡처 목록 누적 패널 | 등록한 요소 개수와 누락 여부를 한눈에 파악 |

**5-3. 기능 흐름**

```text
[화면 2 — 캡처 흐름]
  좌측 브라우저에서 사용자 로그인
       │
       ▼
  예약 화면 요소 클릭 → WebSocket으로 HTML 조각 전송
       │
       ▼
  AI 추론(요소명) → 우측 패널에 "출발역 입력칸" 등 실시간 표시
       │
       ▼
  사용자가 요소명 확인/수정 → 캡처 목록에 추가
       │
       ▼
  필요한 요소 모두 캡처 → [검토] 버튼 → 화면 3
```

**5-4. 레이아웃 구조**

```text
[좌측 메인 60% — noVNC 브라우저 iframe]
[우측 패널 40%]
  상단: 현재 추론 결과 카드 (요소명 / 셀렉터 / 신뢰도)
  중간: [요소명 수정] [캡처 추가] 버튼
  하단: 캡처 목록 (요소명 | 셀렉터 미리보기 | 삭제)
[우측 하단 고정]
  [검토] 버튼 (캡처 1개 이상일 때 활성)
```

**5-5. 주요 컴포넌트** — VncFramePanel, InferenceCard, CaptureList, EditableLabel, PrimaryButton

**5-6. 더미 데이터**

```json
{
  "captures": [
    { "label": "출발역 입력칸", "selector": "#dptRsStnCdNm", "confidence": 0.94 },
    { "label": "도착역 입력칸", "selector": "#arvRsStnCdNm", "confidence": 0.91 },
    { "label": "조회하기 버튼", "selector": "input.btn_inquiry", "confidence": 0.88 }
  ]
}
```

### 화면 3 — 셀렉터 검증·저장 (마스터-디테일)

**5-1. 목적** — 캡처된 셀렉터를 현재 DOM에서 검증하고 최종 저장한다. REQ-ONB-01 세부 3 담당.

**5-2. 기능 + 이유**

| 기능 | 이유 |
|---|---|
| 셀렉터별 매칭 검증 상태 표시 | 저장 전에 깨진 셀렉터를 걸러야 매크로 실패 예방 |
| 상세 패널에서 셀렉터 수동 보정 | AI 오인식 시 사용자가 직접 교정할 수 있어야 함 |
| 전체 저장 + 화면 2 복귀 | 누락·오류 발견 시 캡처 단계로 되돌아갈 수 있어야 함 |

**5-3. 기능 흐름**

```text
[화면 3 — 검증·저장 흐름]
  캡처 목록 자동 검증(현재 DOM 매칭)
       │
       ▼
  좌측 목록에서 항목 선택 → 우측 상세 패널
       │
       ├─ 매칭 OK → 체크
       └─ 매칭 실패 → 셀렉터 수동 입력 → 재검증
       │
       ▼
  모든 항목 OK → [저장] 활성 → 온보딩 완료
  (문제 있으면 [수정]으로 화면 2 복귀)
```

**5-4. 레이아웃 구조**

```text
[좌측 목록 35%]
  셀렉터 목록 (요소명 | 검증 상태 배지: OK/실패)
[우측 상세 패널 65%]
  선택 항목: 요소명 / 셀렉터(편집 가능) / 현재 DOM 매칭 결과 / [재검증]
[하단 바]
  [수정] (화면 2로) | [저장] (모두 OK일 때만 활성)
```

**5-5. 주요 컴포넌트** — MasterList(StatusBadge), DetailPanel, SelectorEditor, RevalidateButton, PrimaryButton

**5-6. 더미 데이터**

```json
{
  "selectors": [
    { "label": "출발역 입력칸", "selector": "#dptRsStnCdNm", "status": "ok" },
    { "label": "도착역 입력칸", "selector": "#arvRsStnCdNm", "status": "ok" },
    { "label": "조회하기 버튼", "selector": "input.btn_inquiry", "status": "failed" }
  ]
}
```
