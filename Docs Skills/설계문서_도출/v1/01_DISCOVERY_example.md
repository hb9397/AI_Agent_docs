## SECTION E — 뒤집기 검증

### 뒤집어진 결정 기록
| 처음 생각 | 바뀐 것 | 이유 |
|-----------|---------|------|
| 로그인도 자동화 | 사람이 직접 로그인 | 보안 + 구현 복잡도 감소 |
| UI 요소 이름을 미리 정의 | AI가 추론, 사람이 확인 | 사이트마다 다르게 적용 가능 |
| 로컬에서만 실행 | Docker 컨테이너 | 이식성 확보 |
| 온보딩은 컨테이너 밖에서만 | 웹 가상화면(:6080)으로 안에서도 | 환경 통일 |

## SECTION F — 최종 결정 요약

### 만들 것
