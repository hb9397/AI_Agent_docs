---

## 1. 한 줄 요약

```
열차 예약 사이트의 UI가 바뀌어도 자동으로 적응하는 예약 자동화 도구
```

---

## 2. 범위

**포함**
- 예약 흐름 자동화 (날짜 / 구간 / 좌석 선택)
- 사이트 UI 변경 감지 및 재학습
- 온보딩 (사용자가 요소를 클릭하면 AI가 역할 추론)

**제외**
- 로그인 자동화
- 결제 자동화
- 모바일 앱

---

## 3. 사용자와 맥락

```
나 혼자 씀. 매주 금요일 오전, 다음 주 열차표 예약.
나중에 가족에게 줄 수도 있으므로 UI는 단순하게.
Docker로 패키징 → 어떤 PC에서도 동일하게 실행.
```

---

## 4. 핵심 흐름

```
[온보딩 — 최초 1회]
1. 사용자가 웹 UI에서 가상 화면(:6080)에 접속
2. 예약 사이트에서 요소(출발역, 도착역 등)를 클릭
3. AI가 각 요소의 역할을 추론 → 사용자가 확인 또는 수정
4. 저장

[매크로 실행 — 반복]
1. 사용자가 날짜 / 구간 입력 후 실행
2. 백그라운드에서 브라우저 자동 제어
3. 완료 또는 실패 알림
```

---

## 5. 데이터

| 이름 | 설명 | 비고 |
|------|------|------|
| site | 등록된 예약 사이트 정보 | URL, 등록일 |
| element | 사이트 내 UI 요소 | 역할명, selector, AI 추론 여부 |
| job | 예약 실행 단위 | 날짜, 구간, 상태(대기/성공/실패) |

---

## 6. 결정된 것 / 결정 안 된 것

| 결정된 것 | 결정 안 된 것 |
|-----------|---------------|
| 로그인은 사람이 직접 | AI 추론 모델: 로컬 vs 클라우드 |
| 온보딩은 가상화면(:6080)으로 | 실패 시 알림 방법 |
| UI 요소 이름은 AI가 추론 | 복수 사이트 지원 시점 |

---

## 7. 제약

```
- 실행 환경: Docker (noVNC 포함)
- 비용: 클라우드 최소화
- 보안: 로그인 자격증명은 저장 안 함
```
