# 사용자 결정 인터뷰 규칙 (decision-interview)

발견 리포트 출력 후 사용자가 항목별로 결정하도록 돕는다.

---

## 인터뷰 원칙

- 모든 항목에 결정을 강요하지 않는다. 보류 가능.
- 사용자 응답 형식을 한정하지 않는다. 자유 텍스트 허용.
- "리포트만 받고 끝" 도 정상 종료 경로다.

---

## 결정 옵션

각 항목에 대해 4가지 중 하나:

| 결정 | 의미 |
|------|------|
| **재사용** | 기존 자산을 import해서 사용. 신규 생성 금지. |
| **확장** | 기존 자산에 옵션/필드/props 추가. 신규 생성 금지. |
| **신규** | 발견된 후보를 사용하지 않고 새로 만든다. (사유 권장) |
| **보류** | 결정 미루기. 패치 산출 대상에서 제외. |

---

## 인터뷰 프롬프트 (리포트 끝에 출력)

```
---

위 발견 결과에 대해 결정을 알려주세요.

자유 형식 예시:
  - "1·2번 재사용, 3번 확장"
  - "전부 신규로 진행, 사유: 도메인이 달라서"
  - "리포트만 받을게요. 패치 안 만들어도 됩니다."
  - "3번만 결정. 나머지 보류."

결정 후 Agent 지시문 패치가 필요하면 "패치 만들어줘" 라고 알려주세요.
패치 없이 종료해도 됩니다.
```

---

## 응답 파싱 규칙

다음 표현을 결정으로 인식한다:

| 표현 | 매핑 |
|------|------|
| "재사용", "reuse", "기존 거 쓸게", "import" | 재사용 |
| "확장", "extend", "props 추가", "필드 추가" | 확장 |
| "신규", "new", "새로", "따로 만들" | 신규 |
| "보류", "나중에", "skip", "패스" | 보류 |
| "전부", "all", "다" | 모든 항목에 일괄 적용 |

번호로 항목을 지목한 경우 (예: "1·2"), 해당 번호 항목에만 적용.

모호한 경우 재질문:
> "{번호}번 항목의 결정이 명확하지 않습니다. 재사용/확장/신규/보류 중 어떤 것인가요?"

---

## 결정 요약 출력

응답을 받으면 다음 형식으로 결정 요약을 다시 보여준다:

```
## 결정 요약

| # | 자산 | 결정 | 비고 |
|---|------|------|------|
| 1 | CreateSite DTO | 재사용 | - |
| 2 | formatKo 유틸 | 재사용 | - |
| 3 | ResultCard 컴포넌트 | 확장 | owner, createdAt 옵셔널 추가 |

확인되시면 다음 중 하나를 알려주세요:
  - "패치 만들어줘" → Agent 지시문 패치 산출
  - "여기까지" → 리포트와 결정만 남기고 종료
  - "{번호} 다시" → 해당 항목 결정 변경
```

---

## 신규 결정 시 사유 권장

사용자가 "신규" 결정 시 **사유를 권장**하되 강요하지 않는다:

> "신규로 진행하시는 사유가 있으시면 알려주세요.
> 패치에 함께 기록해 두면 나중에 코드 리뷰 시 도움이 됩니다.
> (사유 없이 진행해도 됩니다.)"
