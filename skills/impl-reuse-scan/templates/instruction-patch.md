# Agent 지시문 패치 템플릿

사용자가 "패치 만들어줘"를 명시한 경우에만 산출한다.
이 패치는 작업지침서 파일에 자동 반영되지 않는다.
대화창 출력 → 사용자 복사·붙여넣기로 적용.

---

```markdown
# Agent 지시문 보강 패치
**대상**: {Phase N} / {태스크 ID 목록}
**근거**: 공통 자산 발견 리포트 ({YYYY-MM-DD HH:MM} 스캔)

> 아래 내용을 작업지침서의 해당 태스크 "Agent 지시" 섹션 아래에
> 그대로 붙여넣어 주세요.

---

## 재사용 의무 (신규 생성 금지)

<!-- 사용자가 "재사용" 결정한 항목 -->

- **[{태스크 ID}]** `{기존 자산 경로}`의 `{자산 이름}`을 import해서 사용한다.
  - 신규 정의 금지.
  - 시그니처: `{1줄 시그니처}`
  - 기존 사용처: {N}곳 — 변경 시 영향 범위 확인 필요.

- **[{태스크 ID}]** `{기존 자산 경로}`의 `{자산 이름}`을 import해서 사용한다.
  - 신규 정의 금지.

---

## 확장 의무 (신규 컴포넌트/자산 생성 금지)

<!-- 사용자가 "확장" 결정한 항목 -->

- **[{태스크 ID}]** `{기존 자산 경로}`의 `{자산 이름}`을 확장한다.
  - 신규 {컴포넌트/DTO/함수} 생성 금지.
  - 추가할 인터페이스:
    - `{필드 이름}: {타입}` (옵셔널)
    - `{필드 이름}: {타입}` (옵셔널)
  - 기존 사용처 영향 점검: {N}곳
    - `{path:line}`
    - `{path:line}`
  - 옵셔널 추가 방식 권장 — 기존 사용처 변경 없이 호환 유지.

---

## 신규 생성 항목 (의도적 선택)

<!-- 사용자가 "신규" 결정한 항목 -->

- **[{태스크 ID}]** `{생성 예정 자산}` — 신규 생성.
  - 사유: {사용자 입력 사유 또는 "사용자 지정"}
  - 발견된 유사 후보: `{경로}` ({분류})
  - 코드 리뷰 시 위 유사 후보와의 차이를 명시할 것.

---

## 보류 항목

<!-- 사용자가 결정을 미룬 항목 -->

- **[{태스크 ID}]** `{생성 예정 자산}`
  - 발견 후보: `{경로}` ({분류})
  - 결정 전 진행 시 중복 구현 위험.
  - 결정 후 이 패치를 갱신해 적용 권장.
```

---

## 항목별 출력 규칙

### 재사용 항목 필수 정보
- 정확한 import 경로 (상대 또는 모듈 alias)
- 시그니처 1줄
- 기존 사용처 수치 (영향 범위 인지용)

### 확장 항목 필수 정보
- 대상 파일 경로
- 추가할 인터페이스 (필드/props 이름·타입)
- 기존 사용처 영향 점검 안내
- "옵셔널로 추가" 같은 호환 유지 방법 명시

### 신규 항목 권장 정보
- 사용자가 사유를 제공한 경우만 사유 포함
- 발견된 유사 후보를 "비교 대상"으로 명시 (코드 리뷰 도움)

### 보류 항목 처리
- 보류 항목이 있으면 패치 하단에 별도 섹션으로 표시
- "결정 후 패치 갱신 권장" 안내

---

## 빈 패치 처리

모든 항목이 "신규" 또는 "보류"이면 다음과 같이 짧게 출력:

```markdown
# Agent 지시문 보강 패치

이번 결정에서는 재사용/확장 의무 항목이 없습니다.
모든 자산을 신규 생성 또는 보류로 결정하셨습니다.

발견된 후보를 코드 리뷰 시 참고하실 수 있도록
원본 발견 리포트를 함께 보관하시길 권장합니다.
```

## 작업지침서 적용 가이드 (사용자에게 안내할 문구)

패치 출력 후 다음 안내를 함께 표시한다:

```
적용 방법:
1. 위 패치를 복사하세요.
2. 작업지침서 파일을 여세요.
3. 해당 태스크({태스크 ID})의 "Agent 지시" 섹션 아래에 붙여넣으세요.
4. 저장 후 Agent에게 구현을 지시하세요.

본 스킬은 작업지침서를 직접 수정하지 않습니다.
적용은 사용자가 수행해 주세요.
```
