# 멀티 페르소나 코드 리뷰

이 스킬이 호출되면 아래 워크플로우를 즉시 실행하고, 결과를 대화창에 바로 보고하세요. (별도의 .md 파일 생성 금지)

---

## STEP 0 — 플랫폼 및 실행 방식 확인

`prompts/parallel-setup.md`의 [플랫폼 확인] → [모델 목록 표시] → [실행 방식 선택] 절차를 따른다.

병렬 선택 시 아래 Task 목록을 제시한 뒤 `prompts/parallel-setup.md`의 [모델 확정] 절차를 따른다.

| # | Task | 핵심 체크 |
|---|------|----------|
| 1 | Security | 인젝션, 인증 우회, 민감 정보 노출, 권한 검증 누락 |
| 2 | Performance | N+1 쿼리, 불필요한 반복, 메모리 누수, 캐시 미활용 |
| 3 | Maintainability | 과도한 복잡도, 높은 결합도, SRP 위반, 매직 넘버 |
| 4 | Testing | 테스트 누락, 경계값 미검증, 모킹 과다 |

순차 선택 시 아래 리뷰 워크플로우로 직접 진행한다.

---

## 리뷰 워크플로우

1. 특정 파일이나 패키지가 언급 되면 해당 파일 또는 패키지 및 패키지와 연관되거나 하위의 파일들에 대해서 코드리뷰를 진행한다.
2. 특정 파일이나 패키지가 언급되지 않은 경우에는 `git diff`로 변경 파일 및 변경 내용 파악 (unstaged + staged, 커밋 전 전제)
3. 아래 4개 관점의 sub-agent를 Task로 **병렬 실행** — 각 관점별 프롬프트는 `prompts/` 참조
4. 각 관점 결과를 심각도별로 분류·중복 제거
5. 대화창에 종합 리포트 출력 (.md 파일 생성 금지)

| ------------------- | ------------------------------------------------- |
