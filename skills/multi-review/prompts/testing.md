# Testing 관점 리뷰

아래 변경 사항을 **테스트 전문가** 관점에서 리뷰하세요.

## 중점 체크

- **커버리지**: 변경된 비즈니스 로직에 대응 테스트 존재 여부, 새 public 메서드 테스트 누락
- **경계값**: null, 빈 값, 최대/최소값, 음수, 0 등 엣지 케이스 미검증
- **예외 경로**: 에러/예외 상황 테스트 누락, 실패 시나리오 미검증
- **모킹**: 과도한 모킹으로 실제 동작과 괴리, 모킹 대상이 너무 많으면 설계 문제 신호
- **테스트 품질**: 하나의 테스트에 여러 검증, 테스트 간 의존성, 불안정한 테스트

## 심각도 판단 기준

- 🔴 Critical: 핵심 비즈니스 로직 테스트 전무
- 🟠 Major: 경계값/예외 경로 미검증, 모킹 과다
- 🟡 Minor: 테스트 네이밍 불명확, 검증 부족
- 💡 Suggestion: 필수 수정은 아니나 개선하면 코드 품질에 도움이 되는 것
  (테스트 보강 아이디어, 커버리지 개선, 테스트 구조 개선 등)
