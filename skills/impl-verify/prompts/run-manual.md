# 수동 검증 안내 규칙 (run-manual)

manual / hybrid 항목을 사용자에게 체크리스트로 제시하고 응답을 받는다.

---

## 출력 형식

Phase 또는 태스크 단위로 묶어 한 번에 출력한다.

```
## 수동 검증 — Phase 2

검증 환경: {URL / 명령 / 사전 준비 안내}

### 화면 진입
☐ /sites 경로 진입 시 등록 폼 렌더링
☐ 로딩 인디케이터 노출

### 성공 흐름
☐ 유효 입력 → 제출 → 성공 토스트 노출
☐ 목록 화면으로 자동 이동

### 에러 흐름
☐ 필수 누락 → 필드별 빨간 메시지
☐ 서버 500 → 재시도 안내 + 입력 보존

### 접근성
☐ Tab 순서가 폼 → 제출 버튼 → 취소 버튼
☐ 에러 메시지가 스크린리더로 읽힘

답변 형식 예시:
  "전부 PASS"
  "1·2·3 PASS / 4 FAIL (목록 이동이 새로고침처럼 깜빡임)"
  "1 PASS / 2 SKIP (환경 없음) / 나머지 PASS"
```

---

## 사전 준비 안내

검증 환경이 필요하면 출력 상단에 명시:

```
검증 환경:
  - 서버: `npm run dev` 후 http://localhost:3000 접속
  - 테스트 계정: test@example.com / Test1234! (.env.test 참조)
  - 사전 데이터: `npm run seed:dev` 한 번 실행
```

명령은 사용자가 직접 실행한다. impl-verify가 자동 실행하지 않는다(사람 확인 환경이라서).

---

## 응답 파싱

다음 표현을 PASS/FAIL/SKIP으로 매핑:

| 표현 | 매핑 |
|------|------|
| "PASS", "OK", "확인", "정상", "통과" | PASS |
| "FAIL", "안 됨", "안돼", "에러", "실패" | FAIL |
| "SKIP", "패스", "환경 없음", "나중에", "확인 못함" | SKIP |
| "전부", "all", "다" | 모든 항목 일괄 적용 |

번호 또는 항목 키워드로 지목하면 해당 항목에만 적용.

FAIL 응답 시 **간단한 사유**를 함께 요청:
> "{항목} FAIL의 짧은 사유를 알려주세요. (1줄)
>  진단과 권장 액션에 활용됩니다."

---

## 모호한 응답 처리

응답이 불명확하면 재질문:
> "{N}번 항목의 결과가 불명확합니다. PASS / FAIL / SKIP 중 어느 쪽인가요?"

3회 이상 모호하면 SKIP 처리하고 리포트에 기록.

---

## 무응답 처리

사용자가 답하지 않거나 "나중에" 라고 하면:
- 해당 manual 항목 모두 SKIP
- 리포트에 "사용자 미응답"으로 기록
- Phase 게이트 판정 시 "조건부 통과"로 표시

---

## hybrid 항목 전환

hybrid 항목은 다음 순서로 처리:

1. auto 단계 실행 (run-auto 기준)
2. 결과를 사용자에게 보여줌:
   ```
   [hybrid] 스토리북 빌드 완료. http://localhost:6006 에서 확인하세요.
   ```
3. manual 체크리스트 제시
4. 응답 수집

auto 단계가 FAIL이면 manual은 SKIP하고 hybrid 전체 FAIL.

---

## 화면 중심 체크리스트 그룹화

`extract-checks.md`로 추출한 화면 중심 체크리스트는 카테고리별로 묶어 출력:

```
### 기능 검증
☐ {라벨}: {본문}
...

### 상태 검증
☐ {라벨}: {본문}
...

### 에지 케이스
☐ {라벨}: {본문}
...
```

SFR/REQ 태깅이 있으면 항목 앞에 표시:
```
☐ [SFR-005] {본문}
```

---

## 응답 결과 정리

응답을 받으면 다음 형식으로 다시 보여준다:

```
## 수동 검증 결과 — Phase 2

| 항목 | 결과 | 사유 |
|------|------|------|
| 화면 진입 - 폼 렌더링 | PASS | - |
| 화면 진입 - 로딩 인디케이터 | PASS | - |
| 성공 흐름 - 토스트 | PASS | - |
| 성공 흐름 - 목록 이동 | FAIL | 새로고침처럼 깜빡임 |
| 에러 흐름 - 필드 메시지 | PASS | - |
| 에러 흐름 - 입력 보존 | SKIP | 사용자 미응답 |
| 접근성 - Tab 순서 | PASS | - |
| 접근성 - 스크린리더 | SKIP | 환경 없음 |

요약: PASS 5 / FAIL 1 / SKIP 2
```

이 결과가 최종 리포트 생성에 사용된다.
