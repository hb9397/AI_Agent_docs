## 4. FAIL 항목 진단

### ❌ BE-04: pytest tests/sites
- **명령**: `pytest tests/sites/ -v`
- **종료 코드**: 1
- **출력 요약**:
  ```
  FAILED tests/sites/test_create.py::test_validation_missing_name
  AssertionError: expected 400, got 422
  ```
- **추정 원인**: FastAPI/Pydantic 기본 검증이 422를 반환하는데 테스트는 400 기대
- **권장 액션**: 핸들러에서 ValidationError를 400으로 매핑하거나 테스트 기대값을 422로 변경
- **관련 변경**: `M api/routers/sites.py` (5분 전)

### ❌ Phase 2 통합 검증 — 조건 B
- **명령**: `curl -X POST /api/sites -d '{"url":"x"}' -i`
- **응답 상태**: 422 (400 기대)
- **응답 본문**: `{"detail":[{"loc":["body","name"],"msg":"field required"}]}`
- **추정 원인**: 동일 원인 (BE-04 진단 참조)
- **권장 액션**: BE-04 수정과 함께 처리

## 빈 결과 처리

검증 항목이 1건도 추출되지 않으면 다음 형식으로 출력:

```markdown
# 구현 검증 리포트

**작업지침서**: {파일 경로}

검증 항목을 추출하지 못했습니다.
다음 중 하나를 확인하시거나 사용자 입력으로 보완해 주세요:

- 작업지침서에 `**검증 기준**` 또는 `### Phase N 통합 검증` 섹션이 있는지
- impl-doc / impl-fe-be-doc 산출물 형식인지
- 검증 단위 인자가 올바른지 (task=XX / phase=N / full)

수동으로 검증 항목을 입력해 주시면 그대로 검증 매트릭스를 만들어 드립니다.
```

---

## 검증 누락 보강 안내

작업지침서에 검증 기준이 부족하면 다음 한 줄을 리포트 끝에 추가:

```
> 검증 기준이 누락된 태스크가 {N}건 있습니다.
> 작업지침서를 impl-doc / impl-fe-be-doc 으로 다시 한 번 정리해 보강하시길 권장합니다.
> 이 스킬은 작업지침서를 직접 수정하지 않습니다.
```
