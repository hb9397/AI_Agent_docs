# 게이트 판정 규칙 (gate-rules)

검증 결과를 종합해 "다음 Phase 진입 가능 여부"를 권고한다.
권고일 뿐 강제가 아니다. 최종 결정은 사용자.

---

## 판정 단위

### 태스크 게이트

| 조건 | 판정 |
|------|------|
| 태스크의 모든 검증 항목 PASS | ✅ 태스크 완료 |
| 1건 이상 FAIL | ❌ 태스크 미완료 |
| FAIL 없이 SKIP 1건 이상 | ⚠️ 조건부 완료 |

### Phase 게이트

| 조건 | 판정 |
|------|------|
| Phase 내 모든 태스크 완료 + Phase 통합 검증 모든 합격 조건 PASS | ✅ Phase 통과 |
| 태스크/통합 검증 중 1건이라도 FAIL | ❌ Phase 미통과 |
| FAIL 없이 SKIP 1건 이상 | ⚠️ 조건부 통과 |
| Phase 통합 검증 누락 | ⚠️ 통합 검증 누락 |

---

## 권고 메시지

### ✅ Phase 통과

```
Phase {N} 통과 판정.
모든 태스크 검증 + 통합 검증의 합격 조건이 충족됨.
다음 Phase 진입 권장.
```

### ❌ Phase 미통과

```
Phase {N} 미통과.
FAIL 항목 {M}건이 남아 있습니다.
다음 Phase 진입을 권장하지 않습니다.

권장 액션:
  1. FAIL 항목별 진단 섹션 참조
  2. 필요 시 작업지침서 보강 (impl-doc / impl-fe-be-doc 재실행)
  3. 재구현 후 본 스킬로 재검증
```

### ⚠️ 조건부 통과 (SKIP 있음)

```
Phase {N} 조건부 통과.
FAIL은 없지만 SKIP {M}건이 남아 있습니다.
다음 Phase 진입은 가능하나, SKIP 사유를 검토하시고 결정하시길 권장합니다.

SKIP 사유 목록:
  - {항목 1}: {사유}
  - {항목 2}: {사유}
```

### ⚠️ 통합 검증 누락

```
Phase {N}의 통합 검증 시나리오가 작업지침서에 없습니다.
태스크 단위 검증은 통과했으나, Phase 수준의 PASS를 보장할 수 없습니다.

권장 액션:
  1. 작업지침서에 Phase 통합 검증 추가 (impl-doc/impl-fe-be-doc 재실행)
  2. 또는 임시 통합 검증을 본 스킬에서 수동 입력
```

## 게이트 강도 설정

사용자가 명시한 경우 게이트 강도를 조정한다:

| 강도 | 통과 기준 |
|------|----------|
| **strict** | 모든 PASS. SKIP·미응답 모두 미통과로 간주 |
| **standard** (기본) | FAIL 없으면 통과. SKIP은 "조건부 통과" |
| **lenient** | 필수 항목(critical)만 PASS. 그 외는 권고만 |

기본은 standard. 사용자가 "엄격하게" / "느슨하게" 라고 하면 strict/lenient 적용.

---

## 필수(critical) 항목 식별

다음은 강도 무관 필수:

- 빌드 / 타입체크
- 보안 관련 검증 (권한 가드, 인증)
- DB 무결성 검증
- 마이그레이션 적용/롤백

이들 FAIL 시 lenient 모드여도 미통과 권고.

---

## 게이트 출력에 포함할 정보

게이트 권고에 항상 다음을 포함:

1. **판정 (✅ / ❌ / ⚠️)**
2. **수치 요약**: PASS {N} / FAIL {M} / SKIP {K} / 누락 {L}
3. **다음 액션 권장**
4. **재검증 명령** (FAIL 시 어떤 단위로 재검증할지)

예시:

```
## 게이트 판정

| Phase | 판정 | PASS | FAIL | SKIP | 비고 |
|-------|------|------|------|------|------|
| 1 | ✅ 통과 | 8 | 0 | 0 | - |
| 2 | ❌ 미통과 | 5 | 2 | 1 | 통합 검증 합격 조건 2건 FAIL |

다음 Phase(Phase 3) 진입을 권장하지 않습니다.

권장 액션:
  1. Phase 2의 FAIL 항목 진단 참조 (리포트 §3)
  2. 재구현 후 다음으로 재검증:
       impl-verify phase=2
```

---

## 사용자가 게이트 무시 시

사용자가 "FAIL 있어도 다음으로 진행할게요" 라고 하면:

- 강제하지 않는다.
- 다만 다음 한 줄을 리포트에 기록한다:
  ```
  [사용자 결정] Phase {N} FAIL 있음에도 다음 Phase 진입.
  사유: {사용자 입력 또는 "미입력"}
  ```
- 이는 향후 디버깅·추적에 활용된다.
