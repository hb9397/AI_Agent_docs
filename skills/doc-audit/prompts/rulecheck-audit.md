# Task C — 규칙 준수 분석 (rulecheck-audit)

**역할**: 문서에 명시된 금지 규칙·필수 패턴이 실제 코드에서 얼마나 준수되는지 확인하는 전담 에이전트.

---

## 1. 문서에서 규칙 목록 추출

`README.md`, `CLAUDE.md`, `AGENTS.md`, `.instruction/**/*.md` 에서
금지(❌ / 금지 / 절대 / MUST NOT) 및 필수(✅ / 반드시 / MUST) 항목을 추출한다.

## 2. 코드에서 위반 사례 탐지

추출한 규칙을 기반으로 실제 코드를 검색한다.

문서에서 추출한 금지/필수 규칙을 grep 쿼리로 변환하여 실행하라.

## 3. 분석 판단

| 상황 | 제안 방향 |
|------|-----------|
| 위반 사례가 거의 없음 | 현행 규칙 유지 |
| 위반 사례가 1~2건 | 위반 코드 수정 제안 (코드 변경, 문서 유지) |
| 위반 사례가 다수 | 규칙 완화 또는 허용 예외 케이스 문서화 제안 |
| 규칙이 코드 현실과 전면 불일치 | 규칙 재검토 제안 |
