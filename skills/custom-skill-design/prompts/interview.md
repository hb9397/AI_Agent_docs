## [신규] 신규 스킬 인터뷰

### 🔴 필수 확인 (최대 2개 — 반드시 첫 질문으로)

**Q1. 이 스킬이 해결하는 핵심 상황은?**
- Agent가 언제 이 스킬을 꺼내야 하는지 한 문장으로 설명해 달라.
- 예: "코드 리뷰할 때마다", "문서를 최신화할 때", "새 기능 설계할 때"

**Q2. 결과물이 파일인가, 대화창 출력인가?**
- 파일 → 어떤 형식? (md / json / 코드 / docx 등)
- 대화창 → 어떤 구조로? (표 / 목록 / 자유 텍스트)

### 🟡 선택 확인 (필수 답변 후에도 불명확한 경우만)

**Q3. 반복 패턴이 있는가?**
- 매번 비슷한 입력 → 비슷한 출력이면 templates 필요
- 상황마다 다르면 분기 로직 필요

> **원칙**: 한 번에 최대 3개. Q1·Q2 답변으로 충분하면 Q3 생략.
> 추론으로 채울 수 있는 것은 물어보지 않는다.

## [범위 확정] 스킬 범위 확정 체크

아래 항목을 사용자 답변·대화 히스토리·코드베이스에서 추론한다.
추론이 불가능한 경우에만 사용자에게 묻는다.

### 필수 항목

| 항목 | 추론 방법 | 불명확 시 질문 |
|------|----------|--------------|
| 스킬명 | 핵심 동사+명사로 자동 제안 | "스킬 이름을 `{제안명}`으로 할까요?" |
| 핵심 목적 | 사용자 첫 메시지에서 추출 | "이 스킬이 해결하려는 핵심 문제가 맞나요?" |
| 트리거 상황 | 핵심 목적에서 3개 파생 | 확인 게이트에서 함께 검토 |
| 입력 형식 | 대화에서 언급된 파일/데이터 | "입력이 없어도 되나요?" |
| 출력 형식 | Q2 답변에서 확정 | — |
| 필요 도구 | 출력 형식·입력에서 자동 결정 | — |
| **테스트 케이스 필요 여부** | 출력이 객관 검증 가능 → Yes 권장 | "테스트로 검증할까요?" |
| **C-1 확인 단계 필요 여부** | 파일 생성·코드 수정 등 프로젝트 구조 의존 → Yes | — (자동 판정) |

### 테스트 케이스 필요 여부 판단 기준

```
Yes 권장 (객관 검증 가능):
  파일 변환, 데이터 추출, 코드 생성, 고정된 워크플로우

No 권장 (주관 평가):
  글쓰기 스타일, 디자인 품질, 창의적 작업
```

### C-1 확인 단계 필요 여부 판단 기준

```
Yes (프로젝트 구조 의존):
  파일/산출물 생성, 코드 수정, 경로 참조, 빌드/테스트 실행
  → STEP 0에 C-1 확인 단계(단일/복수 프로젝트 감지 + 사용자 확인) 삽입

No (프로젝트 구조 무관):
  대화창 출력만, 외부 API 호출, git 계정 설정 등 독립 유틸리티
  → C-1 확인 단계 생략
```

### 선택 항목 (해당 시에만 확정)

| 항목 | 해당 조건 |
|------|----------|
| 병렬 처리 | 독립 관점 3개 이상 |
| 연계 스킬 | 업·다운스트림 스킬 언급 시 |
| sub-agent | 대규모 병렬 분석 스킬 |

### 도구 자동 결정 규칙

```
파일 읽기 필요   → Read, Glob 추가
파일 쓰기 필요   → Write 추가
코드 실행 필요   → Bash 추가
패턴 검색 필요   → Grep 추가
인터넷 검색 필요 → WebSearch (명시적 요청 시에만)
```
