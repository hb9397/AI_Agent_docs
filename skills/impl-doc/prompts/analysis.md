# 범용 프로젝트 분석 기준 (analysis)

설계 문서를 받으면 출력 전에 아래 항목을 내부적으로 추출한다.

---

## 프로젝트 유형 판별

| 유형 | 판단 기준 | Phase 구성 |
|------|-----------|------------|
| CLI 도구 | 명령줄 인터페이스, args 파싱 | 입력파싱 → 핵심로직 → 출력포맷 |
| 자동화 스크립트 | 크롤러, 배치, 데이터 변환 | 데이터소스 → 처리 → 저장/출력 |
| 라이브러리 / SDK | 외부에서 import해서 사용 | 코어API → 유틸 → 테스트 → 패키징 |
| 단독 백엔드 | API 서버 (FE 없음) | DB+모델 → API → 인증 → 배포 |
| BE 단일 기능 | 기존 BE에 엔드포인트 1~수개 또는 단일 도메인 로직 추가/수정 | DB → 도메인+API → 테스트 |
| FE 단일 기능 | 기존 FE에 컴포넌트/훅/화면 1개 신규·수정 | UI 골격+상태 → 데이터 연동 → 테스트+a11y |
| 데이터 파이프라인 | ETL, 스트림 처리 | 수집 → 변환 → 적재 → 스케줄링 |
| AI Agent / LLM 도구 | 프롬프트, 체인, 에이전트 | 프롬프트설계 → 체인구성 → 도구연동 → 테스트 |
| 인프라 자동화 | Terraform, Docker, CI/CD | 환경정의 → 프로비저닝 → 검증 → 문서화 |

## 추출 항목 체크리스트

### 공통

```
[ ] 프로젝트 유형 (위 표에서 분류)
[ ] 모듈 / 파일 목록 (디렉토리 구조)
[ ] 핵심 로직 식별 (설계 문서 "집중 로직" 섹션)
[ ] 입력 인터페이스 (CLI args, stdin, API 요청, 파일, 환경변수)
[ ] 출력 인터페이스 (stdout, 파일, API 응답, DB, 외부 시스템)
[ ] 외부 의존성 (라이브러리, API, 시스템 명령)
[ ] 설정 관리 방식 (환경변수, 설정 파일, CLI 옵션)
[ ] 테스트 전략 (단위 / 통합 / E2E / 수동)
[ ] 에러 처리 전략 (로깅, 재시도, 폴백)
[ ] 기술 스택 (언어, 프레임워크, 버전)
[ ] 주의사항 / 제약 조건
[ ] 열린 결정 사항
```

### BE 단일 기능 추가 항목

```
[ ] DB 스키마 변경 여부 (신규 테이블 / 컬럼 추가 / 인덱스)
[ ] 마이그레이션 도구 (alembic / prisma / typeorm 등) 및 롤백 전략
[ ] 트랜잭션 경계 (단일 / 다중 리소스 / 분산)
[ ] 인증/인가 컨텍스트 (JWT / 세션 / API Key / 권한 레벨)
[ ] 입력 검증 위치 (DTO / 도메인 / DB 제약)
[ ] 멱등성 요구 (POST 재시도 / 외부 시스템 호출)
[ ] 외부 시스템 사이드이펙트 (메일 / 결제 / 이벤트 발행)
[ ] 응답 스키마 및 에러 코드 (OpenAPI/Swagger 갱신 대상)
```

### FE 단일 기능 추가 항목

```
[ ] 컴포넌트 진입점 (스토리북 / 라우트 / 부모 컴포넌트)
[ ] props / 입력 인터페이스 (타입, 필수/선택, 기본값)
[ ] 상태 종류 (로컬 / 폼 / 서버 상태 / 전역)
[ ] 서버 상태 라이브러리 (react-query / swr / redux / 직접 fetch)
[ ] 4상태 정의 (로딩 / 에러 / 빈 / 성공)
[ ] 인터랙션 시나리오 (클릭 / 입력 / 제출 / 키보드)
[ ] 반응형 브레이크포인트 (모바일 / 태블릿 / 데스크톱)
[ ] 디자인 시스템 / 토큰 의존 (테마, 스페이싱, 컬러)
[ ] 접근성 요구 (WCAG 레벨, 키보드, 스크린리더, 색대비)
[ ] 라우팅 / 권한 가드 (로그인 필요 / 권한별 분기)
```

## Phase 경계 판단 질문

1. 가장 단순한 입력 → 출력 경로 1개는 무엇인가? → Phase 1
2. 설계 문서의 "집중 로직"은 어느 Phase에서 완성되는가? → 핵심 Phase
3. 에러 처리·로깅·모니터링은 별도 Phase가 필요한가? → 규모에 따라
4. 패키징·배포·문서화가 필요한가? → 마지막 Phase
5. 각 Phase가 독립적으로 실행·테스트 가능한가? → 불가능하면 재분할

## 사용자에게 물어볼 우선순위

🔴 반드시 확인:
1. 핵심 로직의 알고리즘/규칙이 충분히 명세되어 있는가?
2. 외부 시스템 연동 인증 방식이 결정되어 있는가?
3. 테스트 전략이 정해져 있는가? (어떤 수준까지 테스트하는가)

🟡 선택 확인:
4. 기존 코드베이스가 있는가? (신규 / 기존에 추가)
5. 배포 환경이 결정되어 있는가? (로컬 / 서버 / 컨테이너 / 서버리스)
