# 병렬 실행 환경 설정 (parallel-setup)
# 역할: 플랫폼 감지 → 실행 방식 선택 → 서브에이전트 모델 확정

---

## 라우팅 표

| 호출 지점 | 실행할 섹션 순서 |
|----------|----------------|
| STEP 0 — 고정 Task 스킬 (multi-review, doc-audit, agent-sync, context-doc) | [플랫폼 확인] → [모델 목록 표시] → [실행 방식 선택] → [모델 확정] |
| STEP 0 — 동적 Task 스킬 (create-prototype, code-comment, rfp-ingest, impl-fe-be-doc) | [플랫폼 확인] → [모델 목록 표시] → [실행 방식 선택 — 선호도만 저장] |
| Task 목록 확정 후 — 동적 Task 스킬 | [모델 확정] |

---

## [플랫폼 확인]

> "현재 어떤 플랫폼에서 실행 중인가요?
> 1) Claude Code  2) OpenAI Codex CLI  3) Gemini CLI  4) 기타 (순차로 진행)"

- 4) 기타 선택 시 → 서브에이전트 없이 순차 실행. SKILL.md의 다음 Step으로 바로 진행.
- 1)~3) 선택 시 → [모델 목록 표시]로 이동.

---

## [모델 목록 표시]

선택한 플랫폼에 해당하는 표만 출력한다.
아래 목록은 2026-06-01 공식 문서 기준 권장 예시다.
실제 사용 가능 여부는 현재 설치된 CLI의 `/model`, `models list`, 설정 파일 등에서 확인한 값을 우선한다.

**1) Claude Code:**
| 모델 | 특징 |
|------|------|
| `claude-haiku-4-5` | 가장 빠르고 저렴 — 서브에이전트 기본 추천 |
| `claude-sonnet-4-6` | 균형형 — 복잡한 작업 권장 |
| `claude-opus-4-8` | 최고 성능 — 비용 높음 |

**2) OpenAI Codex CLI:**
| 모델 | 특징 |
|------|------|
| `gpt-5.4-mini` | 빠르고 저렴 — 서브에이전트용 추천 |
| `gpt-5.5` | 최신 고성능 코딩·추론 — 복잡한 작업 |

| 모델 | 특징 |

## [실행 방식 선택]

> "병렬(서브에이전트)로 실행할까요, 순차로 실행할까요?
> - 병렬: 빠르지만 비용 발생
> - 순차: 느리지만 추가 비용 없음"

- 순차 선택 시 → SKILL.md의 다음 Step으로 진행. 모델 확정 불필요.
- 병렬 선택 시 → [모델 확정]으로 이동.

**동적 Task 스킬 (STEP 0에서 호출):**
- 병렬 선호 시 → 선호도와 플랫폼만 저장. 모델 확정은 Task 목록 확정 후 별도로 진행.
- 순차 선택 시 → SKILL.md의 다음 Step으로 진행.

---

## [모델 확정]

SKILL.md에서 전달받은 병렬 Task 목록을 표시한 뒤 묻는다.

> "서브에이전트 모델을 설정하겠습니다.
> 위 {N}개 Task에 모델을 어떻게 적용할까요?
> 1) 전체 Task에 같은 모델 적용
> 2) Task별로 모델 지정"

- 1) 선택 시 → 사용할 모델 1개 확인. 확정 후 SKILL.md 다음 Step으로 진행.
- 2) 선택 시 → 각 Task별로 모델을 순서대로 수집. 전부 확정 후 SKILL.md 다음 Step으로 진행.
