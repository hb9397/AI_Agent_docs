# 주석 작성 규칙 (comment-rules)

모든 주석은 **한글**로 작성한다.

---

## 0. 변경 범위 한정 원칙

**git diff 또는 mtime으로 변경된 코드가 감지된 경우**, 주석 작성·수정 대상을 아래 기준으로 엄격히 제한한다.

| 대상 | 처리 방식 |
| --- | --- |
| 변경 블록 목록에 포함된 함수·클래스·메서드 | 주석 추가 또는 갱신 |
| 변경 블록 내부의 로직 주석 (분기, 외부 호출 등) | 추가 또는 갱신 |
| 변경 블록 목록에 **포함되지 않은** 함수·클래스·메서드 | **기존 주석 그대로 유지 — 절대 수정 금지** |
| 파일 최상단 블록 주석 | 없으면 새로 작성. 이미 있고 파일 책임이 바뀌지 않았으면 유지. |
| 파일 최하단 변경 이력 | Section 5 규칙에 따라 처리 |

> **판단 기준**: `change-detection.md`의 [변경 범위 추출] 결과에 해당 함수·클래스가 포함되는지 여부로 결정한다.
> 포함 여부가 불분명하면 **수정하지 않는다(보수적 기본값)**.

---

## 0.5. 이모지 사용 원칙

이모지는 **절대 남발하지 않는다.**

허용 대상: 개발자가 반드시 인지해야 하는 중요 경고·주의 지점 1곳에만 사용한다.
- `⚠️` 사이드이펙트, 호출 순서 강제, 전역 상태 직접 변경 등 실수 유발 가능 지점
- `🔒` 인증·권한·보안 관련 처리 (무단 제거 금지 의미 전달)

금지 대상: 파일 요약, 메서드 설명, 변수 주석, 화면 요소, 변경 이력 등 일반 주석에는 사용하지 않는다.
단순 강조나 가독성 목적의 이모지 사용은 금지한다.

---

## 1. 파일 최상단 블록 주석

파일의 가장 첫 줄에 작성한다. 언어별 블록 주석 형식 사용.
아래 4가지 항목을 3~5줄로 서술한다.

- 파일명
- 이 파일이 속한 기능 영역 (단순한 요약이 아닌, 시스템 내에서의 구체적인 책임 영역 명시)
- 이 파일의 역할 (예: API 라우터인 경우 제공하는 주요 API 엔드포인트 목록 등 구체적으로 기능과 역할을 상세히 기재)
- 주요 export / 진입점 (있는 경우)

---

## 2. 클래스 · 메서드 · 프레임워크 요소

대상: `class`, `function`, `const fn = () =>`, React Hook (`useXxx`), Type/Interface, Decorator 등

**상단 블록 주석** (3~5줄)
- 무슨 기능인지
- 어디서 사용되는지 (역할)
- 주요 파라미터 또는 반환값 (있는 경우)

**내부 로직 주석**
- 핵심 분기·처리 흐름을 단계별로 한 줄씩 서술
- 복잡한 조건문·반복문 앞에 의도 설명
- 외부 API·DB 호출 지점에 목적 명시

---

## 3. 변수 · 상수

코드 우측에 인라인 주석으로 짧게 작성한다.
내용이 자명한 변수(`i`, `idx` 등 루프 변수)는 생략해도 된다.

---

## 4. HTML · JSX 화면 요소

대상: `<div>`, `<section>`, `<form>`, `<button>`, `<select>` 등 UI 구성 요소

해당 요소 바로 위에 한 줄 주석으로 서술한다.
- 어떤 화면/섹션을 구성하는지
- 버튼·입력 요소면 어떤 동작을 트리거하는지

---

## 5. 파일 최하단 변경 이력

**신규 파일(git 이력 없음)이거나 주석 추가 외에 다른 실제 코드 변경 사항이 없는 경우** → 파일 하단의 변경 이력 섹션 [최종 수정] 블록 전체를 **절대 작성하지 않고 아예 생략**한다.

**기존 파일** → 최신 변경 1건만 유지 (누적 금지)
- **작성자**: 절대 'Claude AI'나 'AI'로 적지 않는다. 반드시 아래 git 명령어의 결과에서 추출한 실제 작업자(git ID나 이름)를 적는다.
- **변경**: '한글 주석 추가'나 '코드 주석 추가' 등의 내용은 절대 적지 않는다. 명시적인 코드 수정이 있었다면, 반드시 주석을 추가하기 전 마지막에 작업한 **실제 코드 변경 내용**만을 요약하여 적는다.

수집 방법:
```bash
# 마지막 커밋 정보 (작성자 이름, 작성자 이메일, 날짜, 커밋메시지)
git log -1 --format="%an <%ae> %ad %s" --date=format:"%Y-%m-%d" -- <파일경로> 2>/dev/null

# 현재 변경사항 요약
git diff HEAD -- <파일경로> 2>/dev/null | head -60
```

작성자를 git에서 찾지 못하면, 파일에 주석을 반영하기 전 미리보기 단계에서 사용자에게 작성자 이름을 한 번 물어보아 그 값으로 채워넣는다. (절대 AI 이름을 임의로 넣지 않는다.)
